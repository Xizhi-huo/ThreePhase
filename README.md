# ThreePhase

错误设计


错误是怎么被注入的
学员会看到什么现象
应该通过哪一步发现
发现后系统如何反馈
哪些错误允许继续测，哪些必须阻断流程
我建议先把错误场景按 4 类来设计。

一、接线错误类
这类最适合放在前 1 到 3 步暴露。

典型包括：

回路断线
A/B/C 相别接错
PT 二次端子错接
PT1/PT3 与 PT2 对应相接反
Gen2 某相错接到别相
局部缺相
教学价值是：

让学员通过通断测试、线电压测试、相序检查来定位
强化“不是有电就说明接对了”
这类规则要明确：

哪些错误在第一步就能发现
哪些第一步过得去，但会在第三步才暴露
哪些会导致第四步压差异常而不是相序异常
二、工况错误类
这类是“接线没错，但操作状态不对”。

典型包括：

接地方式没切对
Gen1 未建立母排就开始后续步骤
Gen2 已合闸却还在做 PT3 检查
发电机运行中做不该进行的测量
断路器位置在试验位/脱开位却按工作位理解
未开万用表或探针放错点
教学价值是：

强化“测量必须在正确工况下才有效”
训练学员对步骤前提的检查能力
这类错误更适合由流程门禁和测量无效反馈来体现。

三、量值异常类
这类适合放在第 2、4、5 步。

典型包括：

PT 电压偏高/偏低
两侧压差过大
频差过大
相角差过大
电压虽有但不在允许范围
双侧量值不一致但尚未到保护动作程度
教学价值是：

让学员认识“接线正确 ≠ 可以合闸”
训练对同期条件和压差条件的判断
这类错误不应只显示红字，而要让仪表、压差、同步误差条、状态提示一起变化。

四、危险操作类
这类是教学系统里非常重要的一类，因为它体现“错误后果”。

典型包括：

非同期合闸
错相合闸
未完成前置检查强行合闸
运行中错误切换接地方式
带错误接线并入系统后触发大环流/保护
教学价值是：

让学员知道哪些错误不是“测错了重来”，而是“会出事故”
建立操作敬畏感
这类场景建议分两级：

可恢复错误：提示并阻断
事故级错误：允许发生，但给出明显后果和故障分析
接下来设计错误场景，最关键的是不要按“代码实现方便”来列，而要按“教学链路”来列。建议每个错误场景都用统一模板定义：

场景名称
错误注入点
学员可见现象
应该在哪一步被发现
正确判断依据
系统反馈
是否阻断流程
是否允许继续训练
是否需要复位
例如一个标准场景可以写成：

Gen2 B/C 反接
注入点：机组二次接线
现象：第一步可能通过，第二步电压正常，第三步 PT3 相序失败
发现步骤：第 3 步
判断依据：PT3 与 PT2 同名相不匹配
反馈：提示 B/C 相序错误
流程：不得完成第三步
处理：纠正接线后重测
这样后面无论是做 UI、做状态机、做教学文案，都会很顺。

我建议你后面的错误场景设计顺序，不要一口气全铺开，而是分三批：

第一批：最核心、最有代表性的 6 到 8 个错误场景
先覆盖回路、相序、PT、电压、同期。
第二批：操作前提错误
比如接地方式、断路器状态、工作位/试验位、探针位置错误。
第三批：事故级错误
比如非同期合闸、错相并列、保护动作。
